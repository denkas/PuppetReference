Puppet言語: ビジュアルインデックス

このページは文法の名称が思い出せない時、文法要素を探す助けになります。

    file {'ntp.conf':
      path    => '/etc/ntp.conf',
      ensure  => file,
      content => template('ntp/ntp.conf'),
      owner   => root,
      mode    => 0644,
    }

↑ リソース定義.

  • file: リソース型
  • ntp.conf: タイトル
  • path: 属性
  • '/etc/ntp.conf': 値; 例では文字列
  • template('ntp/ntp.conf'): 関数を呼び出すと値を返します; 例では template関数をモジュールにあるテンプレート名を引数として呼び出しています
    package {'ntp':
      ensure => installed,
      before => File['ntp.conf'],
    }
    service {'ntpd':
      ensure    => running,
      subscribe => File['ntp.conf'],
    }

↑ 二つのリソースが使っているbeforeとsubscribeは依存関係のメタパラメータです( リソースリファレンスに記述があります)。

    Package['ntp'] -> File['ntp.conf'] ~> Service['ntpd']

↑ チェイニングアローはリソース間の関係を記述します。リソースリファレンスを参照してください。

    $package_list = ['ntp', 'apache2', 'vim-nox', 'wget']

↑ 変数に配列を代入しています。

    $myhash = { key => { subkey => 'b' }}

↑ 変数にハッシュを代入しています。

    ...
    content => "Managed by puppet master version ${serverversion}"

↑ 予め代入されている組み込み変数をダブルクォートで囲った文字列の中で変数置換しています(例では波括弧を使って変数名を囲っています)。

    class ntp {
      package {'ntp':
        ...
      }
      ...
    }

↑ クラスを定義して、後で使う時にクラスを有効化する事ができます。

    include ntp
    require ntp
    class {'ntp':}

↑ 3つの方法でクラスを宣言する事ができます: include関数による宣言, require関数による宣言、およびリソースライクな宣言があります。クラスを宣言するとクラスに含まれるリソースが管理されるようになります。

    define apache::vhost ($port, $docroot, $servername = $title, $vhost_name = '*') {
      include apache
      include apache::params
      $vhost_dir = $apache::params::vhost_dir
      file { "${vhost_dir}/${servername}.conf":
          content => template('apache/vhost-default.conf.erb'),
          owner   => 'www',
          group   => 'www',
          mode    => '644',
          require => Package['httpd'],
          notify  => Service['httpd'],
      }
    }

↑ 型の定義は新たにリソース型を作れます。例では二つの名前空間を持った名称になっています。

    apache::vhost {'homepages':
      port    => 8081,
      docroot => '/var/www-testhost',
    }

↑ 前述の例で定義したリソース型をを宣言(またはインスタンス化)しています。

    Apache::Vhost['homepages']

↑ リソース参照では事前に宣言したリソースを定義します。各名前空間は常に大文字で始まらなくてはなりません。

    node 'www1.example.com' {
      include common
      include apache
      include squid
    }

↑ ノード定義

    node /^www\d+$/ {
      include common
    }

↑ 正規表現によるノード定義

    import nodes/*.pp

↑ import文はやむを得ない場合を除いて、利用を避けてください。

    # comment
    /* comment */

↑ 2つのコメント

    if str2bool("$is_virtual") {
      warning( 'Tried to include class ntp on virtual machine; this node may be misclassified.' )
    }
    elsif $operatingsystem == 'Darwin' {
      warning( 'This NTP module does not yet work on our Mac laptops.' )
    else {
      include ntp
    }

↑ if文ではagentが設定するfacts変数を使った条件式の結果により条件分岐させています。

    if $hostname =~ /^www(\d+)\./ {
      notify { "Welcome web server #$1": }
    }

↑ if文は正規表現を使えます。regexのパターンマッチ演算子が使えます。

    if 'www' in $hostname {
      ...
    }

↑ if文はin演算子を条件式で使えます。

    case $operatingsystem {
      'Solaris':          { include role::solaris }
      'RedHat', 'CentOS': { include role::redhat  }
      /^(Debian|Ubuntu)$/:{ include role::debian  }
      default:            { include role::generic }
    }

↑ case文

    $rootgroup = $osfamily ? {
        'Solaris'          => 'wheel',
        /(Darwin|FreeBSD)/ => 'wheel',
        default            => 'root',
    }

↑ セレクタを$rootgroup変数に代入する値に使っています。

    User <| groups == 'admin' |>

↑ リソースコレクタは、“spaceship operator”と呼ばれる事もあります。

    Concat::Fragment <<| tag == "bacula-storage-dir-${bacula_director}" |>>

↑ エクスポートリソースコレクタはエクスポートリソースと合わせて使われます。

    Exec {
      path        => '/usr/bin:/bin:/usr/sbin:/sbin',
      environment => 'RUBYLIB=/opt/puppet/lib/ruby/site_ruby/1.8/',
      logoutput   => true,
      timeout     => 180,
    }

↑ リソースのデフォルト設定によりexecリソース型のデフォルトの属性値を設定しています。

    Exec['update_migrations'] {
      environment => 'RUBYLIB=/usr/lib/ruby/site_ruby/1.8/',
    }

↑ リソースのオーバーライドは継承クラスでのみ使う事ができます。

    Exec <| title == 'update_migrations' |> {
      environment => 'RUBYLIB=/usr/lib/ruby/site_ruby/1.8/',
    }

↑ リソースコレクタを使ったオーバーライドはどこでも動きます。危険な手法ですが、稀にとても便利な使い道もあります。

    @user {'deploy':
      uid     => 2004,
      comment => 'Deployment User',
      group   => www-data,
      groups  => ["enterprise"],
      tag     => [deploy, web],
    }
